# Security Policy

## Reporting a vulnerability

Use GitHub private vulnerability reporting for this repository. Do not open a public issue or pull request for a suspected security problem.

If private vulnerability reporting is temporarily unavailable, open a GitHub Security Advisory draft instead of posting details publicly.

## Scope

Report vulnerabilities in:

- core generation tooling
- governance enforcement and hooks
- shipped archetype source and release checks
- repository configuration that changes public distribution or secret-handling posture

## What to include

- affected path or feature
- impact and attacker path
- reproduction steps or proof of concept
- any workaround or mitigation already identified

## Response

Acknowledgement should happen within 5 business days. Fix timing depends on severity, exploitability, and whether the issue affects generated products as well as the core.
